# 正式版迁移计划

本计划基于当前九宫格 Demo 和 `C:\Users\29357\Downloads\gameplay_change_request.md`。
原则：先迁移正式版底座，再逐步实现新增功能，避免把新功能堆在旧 Demo 结构上。

## 已完成

### 第 0 步：清理旧保安亭玩法残留

- 删除旧抽烟进度、基础工资、维修费、破产、旧装备等代码残留。
- 清理旧发电机 / 接线盒占位音。
- 更新 `代码规范.md`，把架构方向改为 2.0 九宫格和统一设备交互接口。

## 迁移阶段

### 第 1 步：正式版存档和关卡进度

目标：把 `Savegame/Game` 从“当前天数”升级为正式关卡进度模型。

- `selected_day`：当前选择并正在游玩的关卡天数。
- `highest_unlocked_day`：已解锁最高天数。
- `cleared_days`：已通关天数列表。
- `best_profit_by_day`：每天历史最高利润。
- `money`：长期货币。
- `owned_equipment`：长期已购装备，正式装备系统接入后使用。
- 新增 `Game.start_day(day)`。
- 新增 `Game.complete_day(completed_day, profit)`。

### 第 2 步：5 分钟经营日和利润账本

目标：把“交够 quota 立刻结算”迁为“固定时长经营日，到点结算”。

- `Ledger.DAY_DURATION`
- `Ledger.time_left`
- `Ledger.profit_today`
- `Ledger.combo_count`
- `Working.update(delta)` 扣时间。
- 时间归零进入 `Settlement`。
- HUD 显示剩余时间、今日利润和连击。
- 当前仍沿用固定产品价值；成本 / 收益 / 连击小费公式在第 4 步实现。

### 第 3 步：选关界面

目标：主菜单进入选关，再选择某天进入主玩法。

- 新建 `day_select.tscn/gd`。
- 显示已解锁天数。
- 显示每日开放设备 / 新机制。
- 显示已通关状态和历史最高利润。
- 提供商店入口，当前先做占位提示，购买逻辑在装备系统底座阶段接入。

### 第 4 步：产品经济重构

目标：产品从固定价值，升级为成本 / 基础收益 / 连击利润模型。

- 已完成 `Product.cost`。
- 已完成 `Product.base_reward`。
- 已完成 `Product.is_processed`。
- 已完成 `Product.is_damaged`。
- 已完成产品出口从自动出货迁为点击按钮生成。
- 已完成正确交付：基础收益 - 成本 + 连击小费。
- 已完成错误交付：扣成本并清连击。
- 已完成结算显示基础收益、成本、连击小费、误交和损坏次数。

### 第 5 步：统一设备交互接口

目标：玩家和猴子使用同一套设备动作接口。

- 设备暴露 id / 类型 / 房间 / 状态 / 可用动作。
- 玩家输入和猴子 AI 都只选择动作。
- 设备提供开始、完成、打断回调。
- 第一批迁入：`ControlPanel`、`ProductExit`、`SelfDestruct`、`PowerBox`、`Heater`。

### 第 6 步：装备系统底座

目标：先建立装备数据、购买和保存，不急着实现完整效果。

- `shock_trap`
- `net`
- 已购买状态保存。
- 运行时数量、最大数量、补充倒计时。
- HUD 显示装备状态。
- 商店能购买并保存。
