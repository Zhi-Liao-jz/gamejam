# 保安亭 · 开发指南

> 一份"先做 Demo、再转正式"的开发约定。读完这份就知道：现在该做什么、什么必须做对、什么可以先糊。

---

## 1. 游戏是什么

**一句话**：你是一名工厂保安，坐在亭里**抽烟摸鱼**就能推进通关进度；但外面有猴子乱搞设备，你**只能靠声音**判断哪里出事，得跑出去驱赶、按手册修设备。烟抽完后按表现结算工资。

**核心张力**（整个游戏好不好玩，只取决于这一句）：
> 亭内安全、能推进度，但**看不见外面、只能听声音**；
> 出门能看清所有设备、能救火，但**不推进度、还可能更糟**。
> 玩家在「摸鱼」和「救火」之间反复纠结。

**一天的循环**：
```
开始 → 抽烟(进度条) → 设备出问题 → 出门维修 → 烟抽完结算 → 商店 → 下一天
						 基础工资 − 维修费 − 设备损失 = 当天收入
```

## 2. 核心策略：Demo → 正式

成败只看一件事：**分清哪些决策"以后改很贵"、哪些"以后改很便宜"**。贵的现在就做对，便宜的随便糊。

### ✅ Demo 阶段就做对（贵，别糊）
| 要做对的 | 为什么 |
|---|---|
| **设备统一接口**（状态 / 可篡改 / 可检查 / 可修复 / 出声） | 以后加接线盒、油箱只是新增一个类，不动框架。最划算的投资。 |
| **模块间用 EventBus 通信**，不直接互相调用 | 猴子→设备→声音→UI 这条链路写死的话，正式版处处要改。 |
| **流程用状态机**（一天的流程、猴子 AI） | 骨架已给（`BaseStateMachine`），顺手用。 |
| **全局状态集中在 `Game` 单例** | 天数、金钱、进度统一一处，存档/重试才好做。 |

### 🔧 Demo 阶段尽管糊（便宜，以后改不亏）
- 数值（工资、维修费、猴子速度）→ 先写死在脚本里，不做配置表
- 美术 → **色块/占位图**，不画正式素材
- 关卡 → 就一天，参数手填
- UI → 能显示数字就行，不排版
- 装备 / 商店 / 多种设备 / 难度曲线 → **MVP（第 1 步）先不做，放到后续步骤**（现已在第 4~5 步完成）；存档留给队友

> 一句话：**接口和通信方式做对，数值和表现尽管写死。**

## 3. MVP 范围（第一个里程碑）

**目标**：一个能从早摸鱼到晚、中间设备会坏、你能修、烟抽完结算工资的 **可玩 1 天**。

**保留**：俯视角移动 · 进出亭 · 抽烟进度条（仅亭内推进）· 一种设备(发电机) · 修复交互 · 当天结算 · 失败/通关判定。

**砍掉**：接线盒、装备、商店、存档、经济评分细节、难度递增、猴子的复杂 AI（先用调试键模拟猴子捣乱）。

## 4. 架构约定

### Autoload（全局单例，见 `project.godot`）
| 名称 | 文件 | 职责 |
|---|---|---|
| `EventBus` | `global/event_bus.gd` | 字符串路由的事件总线。`push_event(目的地, payload)` / `subscribe(目的地, 回调)`。模块解耦用它。 |
| `SoundManager` | `global/sound_manager.gd` | 16 路音效池 + 程序化合成占位音（`_make_tone` / `get_stream`，换真素材填 `SOUNDS` 即可）。设备/猴子用自带 `AudioStreamPlayer2D`（亭内黑屏也按距离衰减+声像可闻）。 |
| `Game` | `global/game_state.gd` | 全局游戏状态 + 配置常量（天数/金钱/进度/今日费用/装备/评分/难度公式），失败通关判定，启动时注册键位。 |

### 状态机
`BaseStateMachine`：子节点即状态（继承 `BaseState`），通过 `transition_to(&"状态名")` 切换。
用于：**一天的流程**（`DayManager`）、以后的**猴子 AI**。

### 设备统一接口 `BaseDevice`（`scripts/devices/base_device.gd`）
所有设备继承它，子类只重写行为：
- 状态：`NORMAL / TAMPERED / FAULT / SEVERE`（正常 / 被篡改 / 故障 / 严重故障）
- `tamper()` 猴子篡改 · `inspect()` 玩家检查 · `repair()` 玩家修复
- 子类重写：`_setup()` `_device_process(delta)` `_on_tamper()` `_on_repair()` `_draw()`
- 状态变化时通过 EventBus 广播 `device_state_changed`（以后声音/UI 订阅它）

### 目录结构
```
global/       autoload 单例 (EventBus / SoundManager / Game)
scripts/
  base_state*.gd        状态机基类
  devices/   base_device.gd, generator.gd ...
  day/       day_working.gd, day_settlement.gd   (一天的流程状态)
  entities/  player.gd ...
  world/     booth.gd ...
  ui/        hud.gd
scenes/    main.tscn (主场景)
assets/    animation/font/image/shaders/sound
```

- [x] **第 5a 步：接线盒 + 失败/通关判定** —— 第二种设备（连线谜题，验证 `BaseDevice` 抽象：猴子/HUD/经济零改自动支持）；致命故障(SEVERE 持续 6s)/破产 → game over，烟抽完无故障 → 通关。

## 6. 当前成果与操作

### 怎么运行
用 Godot 4.6 打开本项目，直接按 ▶ 运行（主场景 = `scenes/main.tscn`）。

### 按键
| 键 | 作用 |
|---|---|
| `WASD` / 方向键 | 移动 |
| `E` / `空格` | 在设备旁：打开控制面板（发电机调参 / 接线盒连线）。**走近猴子即驱赶，无需按键** |
| 面板内 | 拖滑块 / 点端子 / 点按钮调参修复；`E` / `Esc` 关面板（不暂停世界） |
| `N` / `回车` | 结算页 → 进商店；商店页 → 进下一天 |
| `R` | 结算页：重试当天（失败 game over 时只能 `R`） |

### 怎么测核心张力
1. 进保安亭坐下 → 抽烟进度条开始走，画面变黑只剩 HUD。**全程待在亭里**，烟抽完 → 满工资结算。这是"纯摸鱼"基线。
2. 猴子潜入搞设备（亭内只能**靠声音**判断：脚步=有猴来、翻找=在动手、嗡鸣/噼啪=哪台坏了）。
3. 纠结点：**回亭推进度**让损失越积越多、甚至破产/致命 game over；**出门救火**要打开面板调参/连线修好、还可能顾不过来两台设备。
4. 烟抽完结算：`基础工资 − 维修费 − 设备损失 = 当天收入` + 评价；通关后逛商店买装备，下一天难度更高。
