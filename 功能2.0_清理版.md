新版核心循环
1. 产品从左侧房间出口生成。
2. 产品有颜色：红、蓝、绿等。
3. 玩家用 WASD 在九宫格房间之间切换监控画面。
4. 点击产品拿起，再切到目标房间点击放下。
5. 产品需要送到对应颜色交货点，或先送到加工设备处理。
6. 猴子会进入房间，乱按控制面板，甚至尝试打开玻璃罩按自爆键。
7. 玩家需要在多房间之间切换，修正面板、搬货、赶猴子、防自爆。
8. 每天结束后按交货量、损坏、误操作、维修费结算工资，进入商店和下一天。
九宫格地图结构
[ 左上待定 ] [ 上交货点 ] [ 右上待定 ]
[ 产品出口 ] [ 自爆开关 ] [ 右交货点 ]
[ 加热台 ] [ 下交货点 ] [ 发电机/接线盒 ]
颜色建议先数据化，不要写死：
- 上交货点：红色
- 右交货点：蓝色
- 下交货点：绿色
之后可以每天随机颜色位置，作为难度升级。
主要功能模块
1. 监控切房间系统
- 九宫格房间数据。
- WASD 切换当前房间。
- 当前房间高亮。
- 小地图显示九宫格、房间颜色、警告状态。
- 玩家只能直接操作当前监控房间。
- 非当前房间只通过小地图、声音、警报提示异常。
2. 点击拿起/放下系统
- 鼠标点击产品拿起。
- 手上只能持有一个产品。
- 点击地面或交货点放下。
- 切换房间时可以携带产品。
- 当前持有物显示在 UI 上。
3. 产品系统
产品需要有：
- 颜色。
- 当前所在房间。
- 是否被拿起。
- 目标交货点/加工点。
- 价值。
4. 交货点系统
每个交货点包含：
- 颜色标识。
- 接收区域。
- 控制面板。
- 开启/关闭状态。
- 是否被猴子乱按。
- 正确交货加钱。
- 错误交货扣钱或退回。
控制面板建议先这样设计：
面板开启时，交货点正常收货；面板关闭时，产品放上去也不会结算。猴子会
把它关掉，玩家需要切过去重新打开。
5. 产品出口系统
左侧房间生成产品。
- 定时吐出产品。
- 旁边有控制面板。
- 面板开启：正常出货。
- 面板关闭：停止出货。
- 面板被猴子乱按：可能疯狂出货、停止出货、吐出错误产品。
最小版本只做“开/关”。疯狂出货可以作为后期事件。
6. 中央自爆开关
这是主题核心，建议单独作为最高优先级系统。
- 位于中间房间。
- 初始有玻璃罩保护。
- 猴子需要先打开玻璃罩，再按按钮。
- 玩家可以点击关闭玻璃罩、拍走猴子、重置保护锁。
- 玻璃罩状态显示在小地图和 UI 上。
- 如果自爆按钮被按下，进入倒计时。
- 倒计时内玩家可以切到中间房间取消，否则当天失败。
7. 猴子 AI
猴子目标可以分层：
- 低威胁：关闭交货点面板。
- 中威胁：关闭产品出口、乱开出口。
- 高威胁：破坏设备。
- 最高威胁：打开自爆玻璃罩、按自爆键。
猴子行为：
- 从随机房间进入。
- 选择目标。
- 移动到目标房间。
- 操作面板或设备。
- 被玩家点击/装备驱赶。
- 逃跑或短暂眩晕。
最小版本可以先让猴子直接生成在某个房间，然后等待几秒后捣乱。
8. 设备解锁系统
四个角落初始关闭，随天数开放：
右下：发电机 + 接线盒
- 供电系统。
- 被猴子破坏后，部分设备关闭。
- 玩家需要调发电机参数、修接线盒。
左下：加热台，黄色
- 可缓慢处理需要加热的产品。
- 猴子可关闭加热台或调高温度。
- 过热会烧坏产品。
左上、右上待定建议：
- 冷却台：处理蓝色/易融化产品。
- 喷漆台：改变产品颜色。
- 打包机：提高产品价值，但需要时间。
- 分拣器：自动识别颜色，但会被猴子乱设。
- 垃圾压缩机：销毁错误产品，也可能被猴子乱开。
9. 每日流程
- 第 1 天：只有产品出口 + 三个交货点，无猴子或一只慢猴子。
- 第 2 天：猴子会关面板。
- 第 3 天：中央玻璃罩机制出现。
- 第 4-5 天：开放加热台。
- 第 6-7 天：开放发电机/接线盒。
- 后续：更多产品规则、更多猴子、更多设备故障。
